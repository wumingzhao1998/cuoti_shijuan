# Streamlit Cloud 查看部署状态和分支

## 📊 查看部署状态

### 方法1：在应用页面查看（最简单）

1. **访问 Streamlit Cloud**
   - 打开：https://share.streamlit.io/
   - 登录你的账号

2. **查看应用列表**
   - 在主页可以看到你的所有应用
   - 每个应用旁边会显示状态图标：
     - ✅ **绿色圆点**：运行正常
     - 🟡 **黄色圆点**：正在部署中
     - 🔴 **红色圆点**：部署失败或错误

3. **查看详细状态**
   - 点击应用名称进入应用页面
   - 右上角会显示当前状态：
     - "Running" - 运行中
     - "Deploying..." - 正在部署
     - "Error" - 部署错误

### 方法2：在应用设置中查看日志

1. **打开应用设置**
   - 在应用页面，点击右上角 **"⋮"（三个点）**
   - 选择 **"Settings"**

2. **查看部署日志**
   - 在 Settings 页面，点击 **"Logs"** 标签页
   - 这里会显示：
     - **部署日志**：显示构建和部署过程
     - **运行时日志**：显示应用运行时的输出和错误

3. **查看部署历史**
   - 在 Settings 页面，可以看到最近的部署记录
   - 每次 GitHub push 都会触发一次新的部署

### 方法3：查看应用 URL

1. **访问应用 URL**
   - 在应用列表页面，点击应用名称旁边的链接
   - 或者直接访问：`https://你的应用名.streamlit.app`

2. **检查页面状态**
   - 如果页面正常加载：部署成功 ✅
   - 如果显示错误页面：查看错误信息 ❌
   - 如果页面一直加载：可能正在部署中 ⏳

## 🌿 查看使用的分支

### 方法1：在应用设置中查看

1. **打开应用设置**
   - 在应用页面，点击右上角 **"⋮"（三个点）**
   - 选择 **"Settings"**

2. **查看应用配置**
   - 在 Settings 页面的 **"General"** 标签页（默认）
   - 可以看到以下信息：
     - **Repository**：仓库地址（例如：`wumingzhao1998/cuoti_shijuan`）
     - **Branch**：当前使用的分支（例如：`main` 或 `master`）
     - **Main file path**：主文件路径（例如：`app.py`）

### 方法2：在应用列表中查看

1. **查看应用卡片**
   - 在 Streamlit Cloud 主页的应用列表中
   - 每个应用卡片可能会显示分支信息
   - 如果没有显示，需要点击进入应用详情

### 方法3：修改分支设置

如果需要切换分支：

1. **进入应用设置**
   - 应用页面 → 右上角 "⋮" → "Settings"

2. **修改分支**
   - 在 **"General"** 标签页
   - 找到 **"Branch"** 字段
   - 点击编辑按钮（如果有）
   - 选择或输入新的分支名称
   - 点击 **"Save"**

3. **重新部署**
   - 修改分支后，应用会自动使用新分支重新部署
   - 等待 1-2 分钟完成部署

## 🔍 详细步骤：查看你的应用配置

### 完整操作流程

1. **访问 Streamlit Cloud**
   ```
   https://share.streamlit.io/
   ```

2. **登录账号**
   - 使用 GitHub 账号登录

3. **找到你的应用**
   - 在应用列表中找到 `cuoti-shijuan` 或你的应用名称
   - 点击应用名称进入详情页

4. **查看部署状态**
   - 查看右上角的状态指示器
   - 绿色 = 运行正常
   - 黄色 = 部署中
   - 红色 = 错误

5. **查看分支配置**
   - 点击右上角 "⋮" → "Settings"
   - 在 General 标签页查看 **"Branch"** 字段
   - 记录当前使用的分支名称

## ⚠️ 常见情况

### 情况1：代码已推送但应用未更新

**可能原因：**
- Streamlit Cloud 使用的分支与代码推送的分支不一致
- 例如：代码推送到 `master`，但应用配置使用 `main`

**解决方法：**
1. 检查 Streamlit Cloud 使用的分支（Settings → General → Branch）
2. 如果分支不一致，切换到正确的分支
3. 或者在正确的分支上推送代码

### 情况2：部署状态一直显示 "Deploying"

**可能原因：**
- 依赖安装失败
- 代码有错误
- 网络问题

**解决方法：**
1. 查看 Logs（Settings → Logs）
2. 检查错误信息
3. 修复问题后重新推送代码

### 情况3：需要切换到 master 分支

如果你的代码在 `master` 分支，但应用使用 `main` 分支：

1. **方法1：切换分支**
   - Settings → General → Branch
   - 将 `main` 改为 `master`
   - 保存并等待重新部署

2. **方法2：推送代码到 main 分支**
   - 在本地执行：
     ```bash
     git checkout main
     git merge master
     git push origin main
     ```
